# JServer Frontend Development Skill

Этот скилл предназначен для разработки современного фронтенд-приложения для системы JServer.

## 🛠 Технологический стек
- **Framework:** React 18+ (TypeScript)
- **Build Tool:** Vite
- **UI Library:** Рекомендуется **Ant Design** или **MUI** (для готовых компонентов с возможностью переопределения стилей).
- **API Client:** Генерация на основе `openapi.json` (используя `openapi-typescript-codegen` или аналоги).
- **Graph Visualization:** `vis-network.js` (интеграция существующих шаблонов).

## 📂 Ресурсы и шаблоны
В папке `assets/` скилла находятся оригинальные HTML/JS/CSS шаблоны для визуализации графов:
- `assets/two_compare_graph/`: Шаблон для сравнения двух графов (Side-by-side).
- `assets/merged_graph/`: Шаблон для отображения объединенного графа.

При реализации в React необходимо перенести логику из `report_script.js` в React-компоненты, используя `useEffect` для инициализации контейнера `vis-network`.

## 📄 Структура страниц и Роутинг

### Общие страницы
1.  **Авторизация/Регистрация:**
    - `POST /api/auth/register` (уже есть)
    - `POST /api/auth/login` (**ОТСУТСТВУЕТ на бэкенде** — нужно добавить)

### Страницы Администратора
2.  **Лабораторные работы (CRUD):**
    - Список/Просмотр: `GET /api/admin/labs`
    - Создание: `POST /api/admin/labs`
    - Редактирование: `PUT /api/admin/labs/{id}`
    - Удаление: (**ОТСУТСТВУЕТ** — нужно добавить `DELETE /api/admin/labs/{id}`)
3.  **Задания (CRUD):**
    - Список: `GET /api/admin/tasks`
    - Создание: `POST /api/admin/tasks`
    - Редактирование: `PUT /api/admin/tasks/{id}`
    - Удаление: `DELETE /api/admin/tasks/{id}`
4.  **Управление пользователями:**
    - Список студентов: `GET /api/admin/students`
    - Назначение задания: `POST /api/admin/students/{studentId}/assign/{taskId}`

### Страницы Студента
5.  **Мои задания:**
    - Список активных заданий: `GET /api/student/tasks`
6.  **Загрузка и проверка решения:**
    - Страница с деталями задания и формой загрузки ZIP.
    - Загрузка: `POST /api/student/{id}/upload`
    - Кнопка "Проверить": `POST /api/student/{id}/check` -> Переход к сравнению.
7.  **Сравнение графов:**
    - Визуализация результатов `GitComparisonResultDto`.
    - Использование компонентов на базе `vis-network`.

## 🚀 Инструкции по генерации API
1. Получите `openapi.json` (из `target/` после сборки бэкенда).
2. Используйте CLI для генерации хуков или сервисов:
   ```bash
   npx openapi-typescript-codegen --input openapi.json --output ./src/api --client axios
   ```
3. Настройте базовый URL в Axios перехватчике (интерцепторе) для добавления токена авторизации.
