# Проект: Анализ и Сравнение Git-репозиториев

Данное приложение разработано в рамках студенческой работы и предназначено для структурного анализа различий между Git-репозиториев в рамках дисциплины по обучению работы с системами контроля версий.

## Зачем это нужно?
В процессе проверки выполнения студенческих работ по работе с системами контроля версий, в частности с GIT, возникает потребность в сравнении структуры веток или разных версий одного и того же проекта. Данный инструмент автоматизирует процесс сравнения графов коммитов, позволяя:
- Выявлять различия в структуре веток.
- Находить пропущенные или лишние изменения.
- Визуализировать эволюцию кодовой базы.

## Виды сравнения
Приложение поддерживает несколько режимов анализа:
1. **Сравнение в виде двух независимых графов**: Позволяет наглядно сопоставить структуру коммитов двух репозиториев (или двух веток одного репозитория) бок о бок.
2. **Сравнение в виде интегрального графа**: Позволяет увидеть "схлопнутую" структуру, где общие коммиты визуализируются вместе с уникальными для каждой ветки узлами, что облегчает понимание различий.

## Структура проекта
- `JServer/`: Серверная часть (Java, Spring Boot), реализующая алгоритмы сравнения (включая поиск наибольшего общего подграфа).
- `Front/`: Клиентская часть для интерактивной визуализации графов.

## Запуск проекта
Для запуска используйте Docker. В корне проекта находится пример конфигурации:

1. Скопируйте файл примера: `cp docker-compose.yml.example docker-compose.yml`
2. При необходимости настройте переменные окружения.
3. Запустите контейнеры: `docker-compose up -d`
